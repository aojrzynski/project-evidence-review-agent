# Requirements

This synthetic project needs a deployment checklist, rollback plan, support owner, and review notes before a human go-live discussion.

## Operational checks

- Confirm the fake service has a named support rotation.
- Confirm the synthetic deployment checklist has been reviewed.
- Confirm rollback steps are available to the review group.

## Evidence gaps to inspect later

The example pack intentionally leaves some items incomplete so later workflow stages can practice surfacing questions for a human reviewer.
